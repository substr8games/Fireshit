# Fireshit Privacy Policy
## Version 0.1

Fireshit is designed as local software under the user’s control.

## Data practices

The canonical Fireshit suite:

- does not include telemetry;
- does not include behavioral analytics;
- does not use advertising identifiers;
- does not require a cloud account;
- does not automatically upload recordings;
- does not transmit captured application audio or video to Giz@SUBSTR8Games;
- does not sell user information; and
- does not silently send logs to a remote service.

Capture files, configuration, operational state, and logs remain on the user’s system unless the user deliberately sends, synchronizes, publishes, streams, or stores them elsewhere.

## Networked features

A future networked feature must be documented before release. Its documentation must identify:

- destination;
- data transmitted;
- purpose;
- activation method;
- retention behavior;
- whether it is optional; and
- how it is disabled.

A networked dependency introduced by the operating system or an independently installed third-party application is governed by that component’s own policy.

## Capture responsibility

Users control what they capture.

Users are responsible for any authorization, consent, copyright, privacy, employment, contractual, wiretap, and recording-law obligations applying to their use.

Fireshit does not grant rights in third-party applications, displays, communications, images, music, voices, or other content merely because the suite can capture them.

## Support materials

A user who voluntarily submits logs, screenshots, recordings, or configuration for support controls that submission and should remove sensitive material first.

Giz@SUBSTR8Games will use voluntarily submitted support material only for the purpose for which it was provided, unless the user grants additional permission or disclosure is legally required.

## Changes

Material changes to this policy must be documented in the release carrying the change.
